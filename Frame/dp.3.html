# 工厂模式

TIP

工厂模式：工厂模式是用来创建对象的一种最常用的设计模式。我们不暴露创建对象的具体逻辑，而是将将逻辑封装在一个函数中，那么这个函数就可以被视为一个工厂。工厂模式根据抽象程度的不同可以分为：简单工厂，工厂方法和抽象工厂。

# 1.简单工厂模式

简单工厂模式又叫静态工厂模式，由一个工厂对象决定创建某一种产品对象类的实例。主要用来创建同一类对象。

class Phone {
  constructor(name, feat) {
    if(new.target === Phone) {
      throw new Error('抽象类不能实例化!');
    }
    this.name = name;
    this.feat = feat;
  }
}

class XiaoMi extends Phone {
  hasNFC() {
    console.log('我有NFC')
  }
}
class IPhone extends Phone {
  hasA14() {
    console.log('我有A14超强处理器')
  }
}
class HuaWei extends Phone {
  hasPhoto() {
    console.log('我有莱卡超级相机')
  }
}


class Factory {
  create(role) {
    switch (role) {
      case 'xiaomi': 
        return new XiaoMi( '小米手机', 'UI界面好看，系统人性话');
        break;
      case 'iphone':
        return new IPhone( '苹果手机', '系统流畅、性能强悍。');
        break;
      case 'huawei':
        return new HuaWei( '华为手机', '拍照贼溜。' );
        break;
      default:
        throw new Error('参数错误, 可选参数:superAdmin、admin、user')
    }
  }
}

let phoneFactory = new Factory();
let xiaomi = phoneFactory.create('xiaomi');
let huawei = phoneFactory.create('huawei');
let iphone = phoneFactory.create('iphone');

# 2.工厂方法模式

和简单工厂模式中工厂负责生产所有产品相比，工厂方法模式将生成具体产品的任务分发给具体的产品工厂。

class IFactory {
  makePhone() {
    console.log('------抽象工厂------');
  }
}
class XiaoMiFactory extends IFactory {
  makePhone() {
    return new MiPhone;
    console.log('------小米手机------');
  }
}

class AppleFactory extends IFactory {
  makePhone() {
    return new IPhone;
    console.log('------苹果手机------');
  }
}
class MiPhone {}
class IPhone {}

const miFactory = new XiaoMiFactory();
const appleFactory = new AppleFactory();
miFactory.makePhone();
appleFactory.makePhone();

# 3.抽象工厂模式

上面两种模式不管工厂怎么拆分抽象，都只是针对一类产品Phone（AbstractProduct），如果要生成另一种产品PC，应该怎么表示呢？

最简单的方式是把2中介绍的工厂方法模式完全复制一份，不过这次生产的是PC。但同时也就意味着我们要完全复制和修改Phone生产管理的所有代码，显然这是一个笨办法，并不利于扩展和维护。

抽象工厂模式通过在AbstarctFactory中增加创建产品的接口，并在具体子工厂中实现新加产品的创建，当然前提是子工厂支持生产该产品。否则继承的这个接口可以什么也不干。

class IFactory {
  makePhone() {
    console.log('------抽象工厂-生产手机------');
  }
  makePC() {
    console.log('------抽象工厂-生产PC------');
  }
}

class XiaoMiFactory extends IFactory {
  makePhone() {
    return new MiPhone;
    console.log('------小米手机------');
  }
  makePC() {
    return new MiPC;
    console.log('------小米PC------');
  }
  makeMINI() {
    return new MiMINI;
    console.log('------小米mini电脑------');
  }
}

class AppleFactory extends IFactory {
  makePhone() {
    return new IPhone;
    console.log('------苹果手机------');
  }
  makePC() {
    return new MAC;
    console.log('------苹果手机------');
  }
}

class PC {
  constructor() {
    if(new.target === PC) {
      throw new Error('抽象类不能实例化!');
    }
    this.type = 'PC';
  }
}


class Phone {
  constructor() {
    if(new.target === Phone) {
      throw new Error('抽象类不能实例化!');
    }
    this.type = 'Phone';
  }
}

class MiPhone extends Phone {}

class MiPC extends PC {}

const miFactory = new XiaoMiFactory();
miFactory.makePhone();
miFactory.makePC();

# 4.总结

# 工厂方法模式：

一个抽象产品类，可以派生出多个具体产品类。
一个抽象工厂类，可以派生出多个具体工厂类。
每个具体工厂类只能创建一个具体产品类的实例。

# 抽象工厂模式：

多个抽象产品类，每个抽象产品类可以派生出多个具体产品类。
一个抽象工厂类，可以派生出多个具体工厂类。
每个具体工厂类可以创建多个具体产品类的实例。

# 区别：

工厂方法模式只有一个抽象产品类，而抽象工厂模式有多个。
工厂方法模式的具体工厂类只能创建一个具体产品类的实例，而抽象工厂模式可以创建多个。 两者皆可
