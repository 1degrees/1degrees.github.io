# ECMAScript 2015 特性

# 箭头和词法

箭头是使用=>语法的函数简写。它们在语法上类似于c#、Java 8和CoffeeScript中的相关特性。它们同时支持表达式和语句主体。与函数不同，箭头的词法与它们周围的代码相同。如果箭头在另一个函数中，它共享其父函数的“arguments”变量。

// 表达式本体
var odds = evens.map(v => v + 1);
var nums = evens.map((v, i) => v + i);

// 声明式
nums.forEach(v => {
  if (v % 5 === 0)
    fives.push(v);
});

// 词法
var bob = {
  _name: "Bob",
  _friends: [],
  printFriends() {
    this._friends.forEach(f =>
      console.log(this._name + " knows " + f));
  }
};

// 词法参数
function square() {
  let example = () => {
    let numbers = [];
    for (let number of arguments) {
      numbers.push(number * number);
    }

    return numbers;
  };

  return example();
}

square(2, 4, 7.5, 8, 11.5, 21); // returns: [4, 16, 56.25, 64, 132.25, 441]

# Classes 特征

ES2015类是基于原型的OO模式的简单补充。拥有一个方便的声明形式使类模式更容易使用，并鼓励互操作性。类支持基于原型的继承、超调用、实例和静态方法以及构造函数。

class SkinnedMesh extends THREE.Mesh {
  constructor(geometry, materials) {
    super(geometry, materials);

    this.idMatrix = SkinnedMesh.defaultMatrix();
    this.bones = [];
    this.boneMatrices = [];
    //...
  }
  update(camera) {
    //...
    super.update();
  }
  static defaultMatrix() {
    return new THREE.Matrix4();
  }
}

# 增强对象字面量

var obj = {
    // 设置原型prototype. 可以用 "__proto__" 或者 '__proto__' 都可以
    __proto__: theProtoObj,
    // 计算属性名无法设置 prototype
    // 复制 __proto__ 属性.并非原型属性
    ['__proto__']: somethingElse,
    // 等同于 ‘handler: handler’
    handler,
    // 方法
    toString() {
     // Super calls
     return "d " + super.toString();
    },
    // 计算属性名
    [ "prop_" + (() => 42)() ]: 42
};

WARNING

__proto__ 这个特性需要本地支持，并且在以前的ECMAScript版本中已经被弃用了。大多数引擎现在都支持这个特性，但是有些不支持。另外，注意只有web浏览器才需要实现它，因为它在附件b中。它在Node中可用。

# 模板字符串

模板字符串为构造字符串提供了语法糖。这类似于Perl、Python等语言中的字符串插值功能。可选地，可以添加标记以允许定制字符串构造，避免注入攻击或从字符串内容构造更高级别的数据结构。

// 基本文字字符串创建
`This is a pretty little template string.`

// 多行字符串
`In ES5 this is
 not legal.`

// 插入变量绑定
var name = "Bob", time = "today";
`Hello ${name}, how are you ${time}?`

// 非转义的模板字符串
String.raw`In ES5 "\n" is a line-feed.`

// 构造一个HTTP请求前缀用于解释替换和构造

    Content-Type: application/json

# 解构赋值

析构允许使用模式匹配进行绑定，并支持匹配数组和对象。析构是故障软的，类似于标准的对象查找foo["bar"]，在没有找到时产生未定义的值。

// 数组匹配
var [a, ,b] = [1,2,3];
a === 1;
b === 3;

// 对象匹配
var { op: a, lhs: { op: b }, rhs: c } = getASTNode()

// 对象匹配存储
// 绑定到局部变量 `op`, `lhs` 和 `rhs`
var {op, lhs, rhs} = getASTNode()

// 参数位置应用
function g({name: x}) {
  console.log(x);
}
g({name: 5})

// 失效弱化解构
var [a] = [];
a === undefined;

// 使用默认值进行软破坏
var [a = 1] = [];
a === 1;

// 析构+默认参数
function r({x, y, w = 10, h = 10}) {
  return x + y + w + h;
}
r({x:1, y:2}) === 23

# 默认 + 重置 + 扩展

被调用的默认参数值。将数组转换为函数调用中的连续参数。将尾随参数绑定到数组。Rest代替了对参数的需要，并更直接地处理了常见的情况。

function f(x, y=12) {
  // y是12，如果没有传递(或作为未定义传递)
  return x + y;
}
f(3) == 15

function f(x, ...y) {
  // y是一个数组
  return x * y.length;
}
f(3, "hello", true) == 6

function f(x, y, z) {
  return x + y + z;
}
// 将数组中的每个elem作为参数传递
f(...[1,2,3]) == 6

# 变量定义 Let + Const

Block-scoped绑定结构。令为新变量，常量为单赋值。静态限制禁止在赋值之前使用。

function f() {
  {
    let x;
    {
      // 这是可以的，因为它是一个块作用域的名字
      const x = "sneaky";
      // 错误，这里采用的是const定义的块级变量，不容许重新赋值
      x = "foo";
    }
    // 这没问题，因为它是用let声明的
    x = "bar";
    // 错误，已在此块中声明.
    // let , const声明的变量不可以在同一块级作用域内重复声明
    let x = "inner";
  }
}

# 迭代器 Iterators + For..Of

Iterators对象支持自定义迭代，如CLR IEnumerable或Java Iterable。概括了使用for..of完成for..in的自定义基于迭代器的迭代。不需要实现一个数组，支持像LINQ这样的惰性设计模式。


let fibonacci = {
  [Symbol.iterator]() {
    let pre = 0, cur = 1;
    return {
      next() {
        [pre, cur] = [cur, pre + cur];
        return { done: false, value: cur }
      }
    }
  }
}

for (var n of fibonacci) {
  // 将序列截断为1000
  if (n > 1000)
    break;
  console.log(n);
}

WARNING

Iterators是基于这些duck-typed接口(使用TypeScript类型语法只是为了说明):

interface IteratorResult {
  done: boolean;
  value: any;
}
interface Iterator {
  next(): IteratorResult;
}
interface Iterable {
  [Symbol.iterator](): Iterator
}

WARNING

支持通过polyfill

为了使用迭代器，您必须包含Babel polyfill (opens new window)。

# 生成器函数 Generators

生成器使用 function* 和 yield简化迭代器创作。声明为 function* 的函数返回生成器实例。生成器是迭代器的子类型，其中包括附加的next和throw。这使得值能够流回生成器，因此yield是一个返回值(或抛出)的表达式形式。

var fibonacci = {
  [Symbol.iterator]: function*() {
    var pre = 0, cur = 1;
    for (;;) {
      var temp = pre;
      pre = cur;
      cur += temp;
      yield cur;
    }
  }
}

for (var n of fibonacci) {
  // truncate the sequence at 1000
  if (n > 1000)
    break;
  console.log(n);
}

WARNING

生成器接口如下(使用TypeScript类型语法仅用于显示):

interface Generator extends Iterator {
    next(value?: any): IteratorResult;
    throw(exception: any);
}

WARNING

支持通过polyfill

为了使用生成器，您必须包含Babel polyfill (opens new window)。

# 编码 Unicode

支持完整Unicode的非中断添加，包括新的字符串Unicode文字形式和新的RegExp u模式来处理代码点，以及新的api来处理21位代码点级别的字符串。这些新增功能支持用JavaScript构建全局应用程序。

// 与 ES5.1 一样
"𠮷".length == 2

// 新正则表达式行为, opt-in ‘u’, 详情参考MDN官方标准正则表达
"𠮷".match(/./u)[0].length == 2

// 新方式
"\u{20BB7}" == "𠮷"
"𠮷" == "\uD842\uDFB7"

// 新的字符串操作
"𠮷".codePointAt(0) == 0x20BB7

// for-of迭代 code points
for(var c of "𠮷") {
  console.log(c);
}

# 模块语法 Modules

对组件定义模块的语言级支持。从流行的JavaScript模块加载器(AMD, CommonJS)编码模式。由主机定义的默认加载程序定义的运行时行为。隐式异步模型——在请求的模块可用并被处理之前，不执行任何代码。

// lib/math.js
export function sum(x, y) {
  return x + y;
}
export var pi = 3.141593;

// app.js
import * as math from "lib/math";
console.log("2π = " + math.sum(math.pi, math.pi));

// lib/mathplusplus.js
export * from "lib/math";
export var e = 2.71828182846;
export default function(x) {
    return Math.exp(x);
}

// app.js
import exp, {pi, e} from "lib/mathplusplus";
console.log("e^π = " + exp(pi));

WARNING

模块格式器

Babel可以将ES2015模块转换为几种不同的格式，包括Common。js, AMD, System, UMD。你甚至可以创建你自己的。有关更多细节，请参见 babel 模块 (opens new window)。

# Map + Set + WeakMap + WeakSet

常用算法的高效数据结构。WeakMaps提供无泄漏的对象键的副表。

// Sets
var s = new Set();
s.add("hello").add("goodbye").add("hello");
s.size === 2;
s.has("hello") === true;

// Maps
var m = new Map();
m.set("hello", 42);
m.set(s, 34);
m.get(s) == 34;

// Weak Maps
var wm = new WeakMap();
wm.set(s, { extra: 42 });
wm.size === undefined

// Weak Sets
var ws = new WeakSet();
ws.add({ data: 42 });
// Because the added object has no other references, it will not be held in the set

WARNING

支持通过polyfill

为了使用 Maps, Sets, WeakMaps, and WeakSets ，您必须包含Babel polyfill (opens new window)。

# 代理 Proxy

代理允许创建对象，这些对象具有宿主对象可以使用的全部行为。可用于拦截、对象虚拟化、日志记录/概要分析等。

// 代理一个通用对象
var target = {};
var handler = {
  get: function (receiver, name) {
    return `Hello, ${name}!`;
  }
};

var p = new Proxy(target, handler);
p.world === "Hello, world!";

// 代理一个函数对象
var target = function () { return "I am the target"; };
var handler = {
  apply: function (receiver, ...args) {
    return "I am the proxy";
  }
};

var p = new Proxy(target, handler);
p() === "I am the proxy";

# 符号类型 Symbols

符号允许对象状态的访问控制。符号允许通过字符串(如ES5)或符号来键入属性。符号是一种新的基本类型。调试中使用的可选名称参数——但不是identity的一部分。符号是唯一的(如gensym)，但不是私有的，因为它们是通过反射特性(如Object.getOwnPropertySymbols)暴露的。

(function() {

  // 局部symbol类型变量
  var key = Symbol("key");

  function MyClass(privateData) {
    this[key] = privateData;
  }

  MyClass.prototype = {
    doStuff: function() {
      ... this[key] ...
    }
  };

  // 有限的支持来自Babel，全面的支持需要本地实现。
  typeof key === "symbol"
})();

var c = new MyClass("hello")
c["key"] === undefined

WARNING

有限的支持通过polyfill

有限的支持需要Babel polyfill。由于语言的限制，一些特性不能被置换或填充。看到核心。关于更多细节，js的警告部分。

# 内置类型继承

在 ES2015里, 内置类型如： Array, Date and DOM Elements 均可以当做父类实现继承.

// MyArray 继承于 Array父类
class MyArray extends Array {
    constructor(...args) { super(...args); }
}

var arr = new MyArray();
arr[1] = 12;
arr.length == 2

WARNING

部分支持

内置的子类应该在一个案例的基础上评估，如HTMLElement类可以子类化，而许多如Date，Array和Error不能实现，由于ES5引擎的限制。

# Promises

promise是一个用于异步编程的库。Promises是用于未来将要返回一个结果的表示。许多现有的JavaScript库都使用了Promises。

function timeout(duration = 0) {
    return new Promise((resolve, reject) => {
        setTimeout(resolve, duration);
    })
}

var p = timeout(1000).then(() => {
    return timeout(2000);
}).then(() => {
    throw new Error("hmm");
}).catch(err => {
    return Promise.all([timeout(100), timeout(200)]);
})

WARNING

支持通过polyfill

为了支持Promise，你必须包括babel-polyfill。

# 映射 Reflect API

完全反射API，公开对象上的运行时级元操作。这实际上与代理API相反，允许调用与代理目标相同的元操作对应的调用。特别有用的实现代理。

var O = {a: 1};
Object.defineProperty(O, 'b', {value: 2});
O[Symbol('c')] = 3;

Reflect.ownKeys(O); // ['a', 'b', Symbol(c)]

function C(a, b){
  this.c = a + b;
}
var instance = Reflect.construct(C, [20, 22]);
instance.c; // 42

WARNING

支持通过polyfill

为了支持Reflect API，你必须包括babel-polyfill。

# 尾部调用

尾部调用保证不会无限制地增加堆栈。使递归算法在无界输入时更安全。

function factorial(n, acc = 1) {
    "use strict";
    if (n <= 1) return acc;
    return factorial(n - 1, n * acc);
}

// es6之前 栈溢出，
// 但是对于ES2015中的任意输入是安全的
factorial(100000)

WARNING

暂时从Babel 6移除

由于全局支持尾部调用的复杂性和性能影响，所以只支持显式自引用尾部递归。由于其他bug而删除，并将于下个版本babel重新实现。
